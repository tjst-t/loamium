# テンプレートの使い方

決まった形のノート(議事録・読書メモ・日報など)を、**変数を埋めるだけ**で作れる仕組みです。テンプレートの正本はピュア Markdown で、`vault/templates/` 配下に置きます。

> [!tip] このサンプルに同梱のテンプレート
> `make samples` で 3 つのサンプルテンプレートが `templates/` に入ります:
> **議事録** / **読書メモ** / **journal(デイリージャーナル)**。すぐ試せます。

## テンプレートの書き方

`templates/○○.md` を作り、フロントマターの `loamium-template` に設定を書きます。**それ以外のフロントマター + 本文**が「生成されるノートの中身」になります。

```markdown
---
loamium-template:
  description: 会議名・日付から議事録を作成
  target: "議事録/{{date:YYYY}}/{{date:MM}}/{{date:DD}}_{{会議名}}"
  vars:
    - { name: 会議名, type: text, required: true }
    - { name: カテゴリ, type: select, options: [定例, 臨時], default: 定例 }
    - { name: 参加者, type: tags }
tags: [議事録]
created: "{{date:YYYY-MM-DD}}"
---
# {{会議名}}

- 参加者: {{参加者}}

## 決定事項
-
```

- **`target`**: 保存先パスのパターン。`{{変数}}` と `{{date:...}}` を含められます(拡張子 `.md` は自動付与)。省略するとテンプレート名がそのままファイル名になります。
- **`vars`**: 入力させる変数の定義。
  - `name`(必須)/ `type`: `text` / `select` / `date` / `tags`
  - `required: true` … 未入力だと作成できない
  - `options: [...]` … `select` の選択肢
  - `default: ...` … 既定値(モーダルに最初から入る)
- **`description`**: テンプレート選択時に出る説明(任意)。

## 変数の記法

| 記法 | 展開結果 |
|---|---|
| `{{会議名}}` | 入力した変数値 |
| `{{date:YYYY-MM-DD}}` | 対象日(例 2026-07-06) |
| `{{date:YYYY}}` / `{{date:MM}}` / `{{date:DD}}` | 年 / 月 / 日 |
| `{{date:HH}}` / `{{date:mm}}` / `{{date:ss}}` | 時 / 分 / 秒 |
| `{{now:...}}` | 「今」の日時(date は対象日基準) |

- 対応トークンは **YYYY / MM / DD / HH / mm / ss** です(曜日 `ddd` などは未対応でそのまま残ります)。
- `target` に入る変数値は `/` や `..` などが除去され、vault 外に出られないようサニタイズされます。
- **生成されたノートには `{{...}}` やテンプレート記法は残りません**(すべて解決済みのピュア Markdown)。

## 動的選択肢 (optionsQuery)

`vars` の定義に `optionsQuery` を追加すると、選択肢を vault 内のノート集合から動的に取得できます。
静的な `options: [...]` の代わりに使います。

- **`select` + `optionsQuery`** — DQL が返すノートタイトルを候補とする厳格ドロップダウン。候補外の値は拒否されます。
- **`text` + `optionsQuery`** — 候補をサジェストしつつ自由入力も許すオートコンプリート。新規の値も入力できます。

**Epic テンプレートの例** — `#project` タグのノートから親プロジェクトを選ぶ:

```markdown
---
loamium-template:
  description: "Epic ノートを作成する"
  target: "projects/{{プロジェクト名}}/epics/{{Epic名}}"
  vars:
    - name: プロジェクト名
      type: select
      required: true
      optionsQuery: "LIST FROM #project"
    - name: Epic名
      type: text
      required: true
      optionsQuery: "LIST FROM #epic WHERE project = \"{{プロジェクト名}}\""
tags: [epic]
---
# {{Epic名}}

親プロジェクト: [[{{プロジェクト名}}]]

## 概要

## タスク
- [ ]
```

上の例では `{{プロジェクト名}}` が確定した後に、Epic 名の候補が連動して絞り込まれます(**依存クエリ**)。

### optionsQuery の制約

- **値 = ノートタイトル**。DQL `LIST` の結果の title フィールドが候補になります。
- **top-N 50 件**。候補が 50 件を超えると打ち切られます。
- **依存クエリは宣言順**。`{{変数名}}` の参照は宣言順で解決済みの変数のみ有効です。循環参照はエラーになります。
- DQL の文法は [[dataview の使い方]] を参照してください。

## 使い方(UI)

1. サイドバーの「新規ノート ▸ テンプレートから」を選ぶ
2. テンプレート一覧から選ぶ(各項目に保存先プレビューが出ます)
3. **変数入力モーダル**で値を入力(Tab で項目移動・Enter で作成・Esc で中断)。必須が空だと作成できません
4. 作成されたノートがエディタで開きます(保存先が衝突したら自動で連番 `_2` が付きます)

## 使い方(CLI)

REST API・CLI と 1:1 対応です。

```sh
loamium new --template 議事録 --var 会議名=定例会議 --var カテゴリ=定例
#=> 議事録/2026/07/06_定例会議.md に作成
```

## デイリージャーナルのテンプレート化

`templates/journal.md` を置くと、**毎日のジャーナル**(初回アクセス時に自動生成)がそのテンプレートで作られます。生成タイミングは従来どおり遅延生成のまま、テンプレートが無ければ空ファイル(後方互換)です。

```markdown
---
loamium-template: {}
---
# {{date:YYYY-MM-DD}}

## 今日の予定
- [ ]

## メモ
```

## 関連

- 生成ノートの `status` / `rating` などは [[dataview の使い方]] のクエリで横断集計できます。
- 変数入力モーダルの入力ウィジェットは、プロパティUIの意味型(text/select/date/tags)と同じものを使っています。
