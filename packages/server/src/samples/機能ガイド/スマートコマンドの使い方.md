# スマートコマンドの使い方

スマートコマンドは、ノート作成・追記・プロパティ設定・エージェント実行などの定型操作を
1 つの id にまとめた保存済み手順です。繰り返し作業を自動化し、チームや個人の
ワークフローを統一できます。

## 概念

スマートコマンドは `system/commands/` に YAML ファイルとして保存されます。
各コマンドは以下の要素で構成されます:

- **params** — 実行前に入力するパラメータの定義（任意）
- **steps** — 順番に実行するステップの一覧（必須）

ステップには `note-create`（ノート作成）、`journal-append`（ジャーナル追記）、
`template-instantiate`（テンプレート適用）などの種類があります。
ステップ内で `{{パラメータ名}}` を使って入力値を展開できます。

## UI からの呼び出し

Loamium の UI でコマンドパレット（`/` スラッシュメニュー）を開くと、
定義済みのスマートコマンドが一覧されます。コマンドを選択すると
パラメータ入力フォームが表示され、実行するとステップが順に処理されます。

## CLI での操作

### コマンド一覧を確認する

```sh
loamium commands
```

定義済みコマンドの id・名前・説明・必須パラメータが表示されます。

### コマンドを実行する

```sh
loamium command run <id>
```

パラメータを渡す場合は `--param` オプションを使います:

```sh
loamium command run todo-add --param title="設計レビューの準備"
loamium command run meeting-summary --param topic="週次定例" attendees="5"
```

実行後、各ステップの結果（成功・スキップ・失敗）が表示されます。

## 同梱サンプルコマンド

`loamium init-samples` を実行すると、以下のコマンドが `system/commands/` に投入されます:

| id | 名前 | 説明 |
| --- | --- | --- |
| `todo-add` | TODO 追加 | タイトルと優先度を入力して TODO ノートを作成する |
| `log-memo` | ログメモ | 今日のジャーナルにメモを追記する |
| `meeting-summary` | 会議まとめ | 議題と参加者を入力して会議メモノートを作成する |

## コマンド YAML の書き方

コマンドは純 YAML で記述します。`system/commands/<id>.yaml` に配置してください。

```yaml
name: TODO 追加
description: タイトルと優先度を入力して TODO ノートを作成する
params:
  - name: title
    label: タイトル
    required: true
  - name: priority
    label: 優先度
    type: select
    options: [high, medium, low]
    default: medium
steps:
  - kind: note-create
    target: "todos/{{title}}"
    content: |
      # {{title}}

      priority: {{priority}}
      status: open
```

詳しい DSL スキーマは `loamium help command` で確認できます。

## 動的選択肢 (optionsQuery)

`params` の定義に `optionsQuery` を追加すると、入力候補を vault 内のノート集合から動的に取得できます。

- **`select` + `optionsQuery`** — DQL が返すノートタイトルを候補とする厳格ドロップダウン。候補外の値でコマンドを実行しようとすると拒否されます。
- **`text` + `optionsQuery`** — 候補をサジェストしつつ自由入力も許すオートコンプリート。
- **`note` + `optionsQuery`** — タグやフォルダで絞り込んだノートピッカー。全ノートではなく DQL 条件に一致するノートのみを候補にします。

**例 — プロジェクトを選んで Epic を作るコマンド**:

```yaml
name: Epic 作成
description: プロジェクトを選んで Epic ノートを作成する
params:
  - name: project
    label: 親プロジェクト
    type: select
    required: true
    optionsQuery: "LIST FROM #project"
  - name: epic_name
    label: Epic 名
    type: text
    required: true
steps:
  - kind: note-create
    target: "projects/{{project}}/epics/{{epic_name}}"
    content: |
      # {{epic_name}}

      親プロジェクト: [[{{project}}]]

      ## 概要

      ## タスク
      - [ ]
```

上の例では `project` の候補が `#project` タグのノートタイトルから動的に取得されます。

### 依存クエリ

`optionsQuery` 内で `{{変数名}}` を参照すると、上流変数の値が確定してから候補を解決する**依存クエリ**が作れます。

```yaml
params:
  - name: project
    type: select
    required: true
    optionsQuery: "LIST FROM #project"
  - name: epic
    type: note
    optionsQuery: "LIST FROM #epic WHERE project = \"{{project}}\""
    # project が選択されると epic の候補が絞り込まれる
```

### 制約

- **値 = ノートタイトル**。DQL `LIST` の結果の title フィールドが候補になります。
- **top-N 50 件**。候補が 50 件を超えると打ち切られます。
- **依存クエリは宣言順**。参照できるのは宣言順で上位に定義された変数のみです。循環参照はエラーになります。
- DQL の文法は [[dataview の使い方]] を参照してください。
