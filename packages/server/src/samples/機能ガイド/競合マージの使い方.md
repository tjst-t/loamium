# 競合マージ (3-way 自動マージ) の使い方

マルチデバイス・エージェント併用環境で、編集中のノートがリモートで変更されたとき、Loamium は **3-way 差分マージ** を自動実行します。

## 基本動作

| 状態 | リモート変更 | 動作 |
|------|------------|------|
| 未編集 (clean) | 変更あり | 自動リロード (従来通り) |
| 編集中 (dirty) | 別の段落を変更 (非競合) | 自動マージ・カーソル位置を保持 |
| 編集中 (dirty) | 同じ行を変更 (競合) | 競合ダイアログを表示 |

## 非競合の自動統合

ユーザーが段落 A を編集中に、別のデバイスが段落 B を変更した場合:

```
base:   段落 A / 段落 B / 段落 C
ours:   段落 A (ユーザー編集) / 段落 B / 段落 C
theirs: 段落 A / 段落 B (リモート変更) / 段落 C

マージ: 段落 A (ユーザー編集) / 段落 B (リモート変更) / 段落 C
```

エディタには **カーソルを保持したまま** マージ結果が反映されます。ユーザーは編集を続行できます。

## 競合ダイアログ

ユーザーとリモートが **同じ行を異なる内容に変更** した場合、競合ダイアログが開きます。

### 解決方法

各競合ハンクごとに 3 つの選択肢があります:

- **こちらを使う** — ローカルの編集内容を採用
- **リモートを使う** — リモートの変更内容を採用
- **両方保持** — ローカル編集の後にリモート変更を付加

全ハンクを解決すると「マージ結果を保存」ボタンが有効になります。

### キャンセル

「後で解決」でダイアログを閉じると、ローカル編集は保持されたまま競合は未解決になります。次にリモート変更があれば再度ダイアログが表示されます。

## 注意事項

- **保守的競合検出**: 誤マージを防ぐため、曖昧なケースは競合として扱います。本当に競合していなくても競合ダイアログが出ることがあります。
- **競合マーカー不使用**: ファイルには `<<<`, `===`, `>>>` などの競合マーカーは書き込まれません。ダイアログ上で並列表示します。
- **エージェントの書き込みは別軸**: エージェントによる書き込みは 3-way マージを経由せず、last-write-wins (楽観ロック) で処理されます。

## 仕組み (技術的詳細)

3 つのバージョンを比較します:

- **base**: 最後にサーバーから取得したリモート内容 (セッション内のみ保持、揮発)
- **ours**: エディタの現在の編集バッファ
- **theirs**: リモートの最新内容 (SSE `notes_changed` 受信後に取得)

`diff3Merge(base, ours, theirs)` で行単位の 3-way マージを実行し、非競合ハンクは自動統合します。
