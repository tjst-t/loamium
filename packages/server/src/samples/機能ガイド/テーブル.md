# テーブル

標準の Markdown テーブル(GFM)がそのまま使えます。カーソルがテーブル外にあると **WYSIWYG 表示**になります。

| 機能 | 操作 | 補足 |
| --- | --- | --- |
| セル編集 | セルを 1 クリック | そのまま入力できます |
| セル移動 | Tab / Shift+Tab / Enter | Excel 風。行末の Tab は次行へ |
| 行を追加 | 表の下端のバー(ホバーで表示) | 最終セルで Tab でも追加 |
| 列を追加 | 表の右端のバー(ホバーで表示) | |
| 行/列を削除 | セル右上・行右端の × | |
| ソース編集 | 表右上の「ソースを編集」 | `\|` 記法の生テキストに戻る |

## 揃え(アライメント)

区切り行の `:` で列の揃えを指定できます。

| 左揃え | 中央 | 右揃え |
| :--- | :---: | ---: |
| left | center | right |
| ひだり | まんなか | みぎ |

## 幅の表示モード(標準 ⇔ 収める)

読み取り列(約 820px)より広いテーブルの表示方法を、テーブルごとに切り替えられます。テーブルにマウスを重ねると**左上に幅トグルボタン**が出ます。

- **標準**(既定): 列は自然な幅のまま。読み取り列より広い場合は**テーブル内だけ横スクロール**します(本文レイアウトは動きません)。
- **収める**: テーブルを読み取り列の幅に**強制的に収め**、セルの内容を折り返します(2 行以上になっても全列が一度に見えます)。

> [!note] 表示状態はノートに書かれません
> 幅モードは表示専用の設定で、ブラウザ(localStorage)に保存されます。Markdown ファイルには一切書き込みません(ピュア Markdown 絶対)。ヘッダや列数を変えると既定(標準)に戻ります。

## ポイント

- 編集結果は常に**標準 Markdown テーブル**として保存されます(独自記法なし)
- セル内の `|` は `\|` に自動エスケープされます
- 新しいテーブルは [[スラッシュメニュー]] の `/table`(サイズピッカー付き)で挿入できます
