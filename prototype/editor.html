構成図

容量見積もり
スナップショット保持数 n = 14 のとき、必要容量は次式で見積もる:
Ctotal = Cbase + Σi=1n ΔCi × (1 + rmeta)
直近の実測では ΔC ≈ 3.2 GB/日 なので、512 GB で約 4 ヶ月分。
移行スクリプト
bashshiki: github-light
# VM イメージを NAS へ退避してから移行する for vm in $(qm list | awk 'NR>1 {print $1}'); do vzdump "$vm" --storage nas-backup --mode snapshot done
移行当日は 2026-07-03 のジャーナルに手順ログを残すこと。
PROTO レンダラーは fence / inline / block の 3 レジストリ経由で登録される(AC-S9ab6c3-2-3)。 mermaid = fence(replace)、コードフェンス = fence(augment)、$…$ = inline、$$…$$ = block。 カーソル行(上の「3-2-1 ルール」の行)だけ装飾が外れ、Markdown ソースが見える。
