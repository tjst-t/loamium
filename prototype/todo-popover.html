PROTO 新データモデル: チェックボックス - [ ]/- [x] は done/not-done のみ (plain GFM)。status/priority/due は OPTIONAL インラインフィールド。
シンプルタスク: 「牛乳を買う」「スプリントレトロスペクティブ…」はフィールドなし → ピル一切なし。リッチタスク: インラインフィールドあり → 対応ピル表示。
アクティブ行 (行 8): ソース表示 — [status:: progress] [priority:: high] [due:: 2026-07-25] が見える。
ポップオーバー: ステータス (なし/クリア + 語彙から4オプション) + 期日 (今日/明日/来週/月グリッド/クリア) + 優先度 (語彙 + なし)。done 選択 → checkbox [x] に同期 (逆も同様)。
ストレージ: POST /api/notes/{path}/patch。ステータス/優先度の語彙は system/settings.yaml で定義(コード埋め込みなし)。
