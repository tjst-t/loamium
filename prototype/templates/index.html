テンプレートシステム

vault 内の可視 templates/ 配下 Markdown をそのまま「テンプレート」として扱う汎用機構。 テンプレートのフロントマター loamium-template.target(保存先パターン)と vars(ユーザー変数定義)で駆動し、 {{name}} と {{date:FORMAT}} を解決して新規ノートを作成します。 新規作成時に変数入力モーダルを出し、キーボードで完結します。

テンプレート選択(新規作成導線)

サイドバー「新規ノート ▸ テンプレートから」/ コマンドから開くテンプレート一覧。議事録 / デイリー / 読書メモ / プロジェクトの各項目に、 いま作るとどこへ保存されるか(例 → 議事録/2026/07/06_{{会議名}}.md)のプレビュー付き。選択で変数入力モーダルへ。

変数入力モーダル(最重要)

「議事録」テンプレートの vars に応じた入力欄(会議名=text必須 / 日付=date / カテゴリ=select / 参加者=tags)。 property-types と同じ入力ウィジェットを流用。Tab 移動・Enter 確定・Esc 中断のキーボード完結、必須未入力のインライン検証、 上部に解決後の保存先ライブプレビュー。初期 / エラー / 入力済みの 3 状態を併記。

template-modal / template-target-preview / template-create
要点 — 保存先パターンと解決

保存先パターン(target)→ 解決例

テンプレート「議事録」のフロントマター:

target: /議事録/{{date:YYYY}}/{{date:MM}}/{{date:DD}}_{{会議名}} 今日 = 2026-07-06 / 会議名 = 定例会議 → 議事録/2026/07/06_定例会議.md

{{name}} は vars で定義したユーザー変数、{{date:FORMAT}} は今日の日付(モーダルの「日付」変数で上書き可)。 ファイルには独自記法もブロック ID も書かず、生成物はピュア Markdown のまま。

{{date:FORMAT}} 早見表

2026-07-06 09:04:20(月)を例に:

FORMAT意味解決値
YYYY西暦 4 桁2026
MM月(0 埋め)07
DD日(0 埋め)06
HH時(24h)09
mm分04
ss秒20

組み合わせ({{date:YYYY-MM-DD}} 等)も可。未定義の変数や不正な FORMAT はモーダルで検証され、確定できません。
