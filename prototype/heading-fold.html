Loamium 設計概要
Loamium はローカル Markdown を正本とする個人用ノートアプリ。アウトライナー編集(C 方式)とエージェント統合を両立する。
アーキテクチャ
TypeScript strict / Node.js 22 / npm workspaces モノレポ構成。
フロントエンド
• React + CodeMirror 6 (lezer-markdown)
• ライブプレビュー: Decoration API でレンダー
• アウトライン操作 (C 方式)
◦ Tab でインデント / Shift-Tab でアウトデント
◦ 見出し行では Tab を消費しない
バックエンド
• Hono (REST API)
• ファイルシステム読み書き (Node.js fs/promises)
• Fuse.js (全文検索)
• zod (スキーマ検証)
API エンドポイント
• GET /api/notes — ノート一覧
• GET /api/notes/:path — ノート取得
• PUT /api/notes/:path — ノート保存
• PATCH /api/notes/:path — リネーム
• DELETE /api/notes/:path — 削除
開発フロー
1. make serve でバックグラウンド起動
2. make test で全ワークスペーステスト
3. make lint で型チェック + lint
テスト戦略
• Vitest — ユニットテスト(パース・リンク解決・API)
• Playwright — E2E / モック UI テスト
• JUnit XML を reports/ に出力
設計原則
ピュア Markdown 絶対 — ブロック ID・独自記法をファイルに書き込まない。
vault 内パスは必ず packages/shared の正規化ユーティリティを経由する。
PROTO 見出し折りたたみ (Sb6f1d3-1):
  • 各見出し行の左ガターに heading-fold-toggle が表示される (hover で表示、折りたたみ中は常時表示)
  • 「バックエンド」 (H3) のセクションが折りたたみ中 — toggle が folded クラス + chevron が -90° 回転し、fold-pill 「… 12 行」が表示される
  • 既存のリスト fold-toggle (「アウトライン操作」行のガター) と 共存 — ガター列は別 (.gutter で統合。実装は別 GutterMarker)
  • 折りたたみ状態は ファイルに書き込まれない (セッションのみ / リロードでリセット)
  • キー: Ctrl-Shift-[ で折りたたみ、Ctrl-Shift-] で展開
