ウィジェット比較 — 動的 select / autocomplete / 静的 select の違い
要点 動的 select (data-widget="dynamic-select"): モーダル表示時に POST /api/options-query で候補を取得してから <select> に挿入する。取得中は disabled + スピナー。0 件は <input type=text> に切り替えてフォールバック。任意入力は不可 — 候補のいずれかしか選べない(静的 select と同じ制約)。

autocomplete (data-widget="autocomplete"): <input> + <datalist> によるネイティブサジェスト。候補が来なくても 自由入力が常に可能。候補取得はノンブロッキングで、入力欄はモーダル表示直後から使える。IME 変換中は誤確定を防ぐ。

静的 select (既存・比較用): options[] をテンプレート定義で固定。ローディングなし。ノートの増減には追従しない。

note-picker (data-widget="note-picker"): コマンド param の type: note 専用。DQL で候補を絞り込んだノートリストを表示し、自由入力(テキストマッチ)も可。依存クエリに対応 — 上流変数が確定すると resolvedVars を注入して再フェッチする。
