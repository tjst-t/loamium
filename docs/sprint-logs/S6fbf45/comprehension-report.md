# Comprehension Report — M2: UI MVP 完成 (Sprints Sa704c3, S9ab6c3, S6fbf45)

_Generated at milestone arrival. Read this before `autopilot review`._

## What changed

- **ブラウザで完結するノートアプリになった。** `make serve`(API)+ `make serve-ui`(UI)で、3 ペイン構成(ファイルツリー / CodeMirror エディタ / バックリンクパネル)の UI が動く。起動すると今日のデイリージャーナル(自動生成)が開き、前日/翌日ナビと一覧から過去のジャーナルへ移動できる。
- **編集して保存できる**: Cmd/Ctrl+S と 1.5 秒デバウンスの自動保存。ツリーから新規ノート・フォルダ作成・リネーム・削除。保存はピュア Markdown のまま(E2E で実ファイルをバイト検証)。
- **同時編集の後勝ち事故に警告が付いた**: 保存時に mtime ベースの競合検出(サーバーが 409 を返し、UI が競合ダイアログを出す)。外部プロセスが作ったノートを新規作成で silent 上書きすることも防止。
- **アウトライナー編集(C 方式の核心)**: リスト行(`-` / `1.` / `- [ ]`)でのみ Tab/Shift+Tab のインデント(子要素追従)、ガターからの折りたたみ、チェックボックスのクリックトグル(`- [x]` としてファイル保存)。見出しや段落では Tab は普通のタブのまま — 文章ノートを歪めない。
- **ライブプレビュー**: カーソル行以外は見出しサイズ・太字・インラインコード・[[リンク]] が装飾表示され、カーソルを置いた行だけソースが見える(Obsidian Live Preview 相当)。mermaid 図・KaTeX 数式・Shiki コードハイライトが実描画される(すべてローカルバンドル、オフラインで動く)。
- **拡張は 3 レジストリ経由**: fence / inline / block のレンダラー登録機構が実装され、mermaid・KaTeX・Shiki はその上に乗っている。新しい記法は登録 1 件で追加できる(ユニットテストで検証済み)。
- **リンク機構が完成**: `[[` と打つと既存ノート名の候補(部分一致)が出て選択挿入。存在しないノートへのリンクは赤+破線で表示され、クリックで新規作成。リンククリックでノート間を移動。バックリンクパネルに参照元とコンテキスト行が表示される。
- **リネームがリンクを壊さなくなった**: ノートをリネームすると vault 内の全 `[[旧名]]`(`[[旧名#見出し]]` / `[[旧名|表示名]]` 含む、コードフェンス内は除外)が自動で書き換わる。API(POST /api/notes/{path}/rename)と CLI(`loamium rename`)からも同じことができる。

## Why this way

- **プロトタイプがビジュアルの正**: 実装前に `prototype/` の静的 HTML を生成し、DOM 構造と data-testid 契約(prototype/TESTIDS.md)を先に固定してから実装した。※ユーザー不在の自律実行だったためプロトタイプは未承認のまま使用 — 今回のレビューで実 UI と合わせて確認してほしい。
- **インデント単位は 4 スペース、先頭項目の字下げは不可**: E2E が「先頭リスト項目をインデントすると CommonMark 上コードブロック化してピュア Markdown が壊れる」ことを検知し、アウトライナー標準(直前に兄弟がある行のみインデント可)に修正した(priority 1: 正本性)。
- **rename は compute-then-apply**: 全書き換えを計算し終えるまでディスクに一切書かない。曖昧な同名 basename リンク(解決先が旧パスでないもの)は書き換えない。インデックスではなく FS を直接走査(priority 6: ファイルが正)。
- **`loamium rename` を CLI に追加**: SPEC の CLI 表に rename は無いが、「API と CLI は 1:1」(priority 3)を優先して追加した。
- **mermaid/KaTeX/Shiki は npm 同梱**(CDN 不使用): オフラインで動くことを優先。バンドルサイズ警告は許容(priority 5: シンプルさ)。
- **portman 適合**: この環境の portman は `acquire`/`port` 未対応だったため、Makefile を `portman lease` ベースに再実装し CLI にフォールバックを追加(環境適合。CLAUDE.md に反映済み)。

  - (S9ab6c3, verifier 指摘) インラインレンダラーの失敗が無言 catch だった → console.warn 追加で解消。
  - (S9ab6c3) Shiki の対応言語は固定リスト約 30 件。未登録言語はソース表示フォールバック。

  - (S6fbf45, verifier 指摘・overlooked) rename 後のインデックス再構築失敗は console.error のみで続行(ファイルは正しく書き換わっており、インデックスは自己修復可能)→ decisions.json に記録済み。

## What was assumed

- UI のデザイントーン(ライト基調・アクセント紫 #6e56cf・Obsidian/Linear 風)はプロトタイプ生成時の自律判断。ダークモードは未実装(スコープ外扱い、設定 UI はプレースホルダ)。
- 自動保存のデバウンスは 1.5 秒、インデント単位は 4 スペース(いずれも設定化していない)。
- 保存進行中(数 ms)のノート切替クリックは no-op(連打時の整合性を優先)。
- `[[リンク]]` 挿入時の表記は「最短一意」形式(basename が一意なら basename、重複があればフルパス)— 書いたリンクが必ず解決することを優先。
- モバイル表示は最低限のレスポンシブのみ(VISION どおり閲覧割り切り、本格対応はスコープ外)。
