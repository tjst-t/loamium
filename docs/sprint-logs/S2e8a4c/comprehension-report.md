# Comprehension Report — マイルストーン: ロードマップ 100% (S7e2d5c / Sa6c3b0 / Sd5c9f4 / Se3b7a2 / Sb6f1d3 / S2e8a4c)

_マイルストーン到達時に生成。`autopilot review` を始める前にこのファイルを読んでください。_

## How to run it

```sh
make serve
```

起動後、`portman lease --name loamium` で取得したポートをブラウザで開く(通常 `http://localhost:5173`)。dev vault は `dev-vault/` に自動生成。サンプルを投入したい場合は `loamium init-samples` を別途実行(または `make samples`)。

## Why this way

**タスクデータモデル: ADR-0029 (Se3b7a2)**

プロトタイプレビューでユーザーが「alt-checkbox(`- [/]`/`- [!]`)は前知識がないと分かりづらい、単純チェックボックスには過剰」と指摘。代替案 3 択(alt-checkbox 継続 / LogSeq 行頭キーワード / Dataview インラインフィールド)を検討し、Dataview インラインフィールド方式を採用した。理由は「チェックボックスは GFM 標準の完了/未完了のみに戻し、段階的な複雑さを持てる」「Obsidian Dataview と互換でプレーンテキストとして無傷」「語彙を YAML で設定できコードに enum を埋め込まない」の 3 点。alt-checkbox と LogSeq 式は ADR-0029 にて明示的に不採用記録。

**DQL キャッシュ: インメモリのみ (Sd5c9f4)**

「インデックスは使い捨て・ファイルが正本」(DESIGN_PRINCIPLES priority 6 / ADR-0001)に従い、永続化なし。サーバー再起動で再生成できる設計を AC 化。Redis 等の外部ストアは導入しない。

**SSE エンドポイントにエージェントツールを追加しない (Sd5c9f4)**

SSE は UI の受動購読インフラであり、エージェントの「操作」対象ではない(エージェントはサービス層を直接呼んで最新結果を得る)。ADR-0014(常時=base / 詳細=help)の観点でも help トピック不要と判断。

**フォルダ移動 = 配下ノートの逐次 rename (S2e8a4c)**

専用 move エンドポイントを新設せず、既存の `POST /api/notes/{path}/rename` を再利用(ADR-0016 監査済みサービス層経由)。ウィキリンク追従はサーバー側 `renameNote` が担保する。トランザクションなしの逐次 PATCH は部分移動リスクを持つが、エラー時はトースト通知で止まる設計を選択(priority 5 シンプルさ)。

**見出し fold: セッション中のみ、.md 非書き込み (Sb6f1d3)**

`- fold 状態を `.md` やストレージに永続化しない」を AC 化。EditorState による自然なリセットで副作用ゼロ(DESIGN_PRINCIPLES priority 1 ピュア Markdown 正本)。

**TOC クリック → unfold 連携が window グローバル経由 (Sb6f1d3)**

当初設計は `onViewReady` コールバック + App の `editorViewRef` で完結させる予定だったが、実装上 `window.__loamiumEditorView__` / `window.__loamiumUnfoldAtCursor__` グローバルになった。型付き・any 不使用・動作正常・テスト green ではあるが、設計上の負債として `compromises.json` に記録。refine 時に修正を推奨。

## What was assumed

- **SSE 1 秒反映 e2e のタイムアウト**: 「1 秒以内に反映」の受け入れ基準に対し、CI 安定性のため e2e タイムアウトを 2 秒に設定している。実環境で 1 秒を大きく超える場合は debounce/全件再取得フォールバックの検討が必要。
- **フォルダ移動のアトミック性**: 配下ノートの逐次 `renameNote` はトランザクションなし。途中失敗時の部分移動はトーストのみで止まる。大規模フォルダ移動の要件が出た場合は専用ロールバックロジックが必要。
- **タスク語彙の未設定時フォールバック**: `system/settings.yaml` の `tasks:` セクションが未設定の場合、組み込みデフォルト語彙(statuses: todo/progress/blocked/done、priorities: highest/high/medium/low)が使われると仮定している。ユーザーが空語彙にしたい場合のフォールバック挙動は要確認。
- **チェックボックスと status の同期**: `done:true` フラグを持つ status を選択するとチェックボックスが `[x]` に同期する設計だが、逆方向(チェックを入れると status が自動変化する)も含めた全パターンの整合は e2e でカバーされている前提。目視で端境ケースを確認推奨。
- **PWA のスコープ**: `manifest.json` + SW の構造は実装・テスト済みだが、Lighthouse 90+ の数値は本番 HTTPS ビルドでのみ計測可能。CI では構造的正確さのみを保証する。
- **モバイル閲覧中心**: スマホでの CodeMirror 入力は動作するが、専用モバイルエディタ体験(ツールバー・仮想キーボード対応・スクロール同期)はスコープ外と仮定した(VISION「外出先は閲覧中心」)。本格モバイル編集が必要なら別スプリントで対応。
