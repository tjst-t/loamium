# Comprehension Report — 内蔵エージェント第1段 (Sprint S53409d)

_Generated at milestone arrival. Read this before `autopilot review`._

## What changed

- **右パネルの「Claude」タブ(ターミナル)が「エージェント」タブに置き換わった。** 以前は node-pty で `claude` CLI を生やす端末を貸すだけだったが、いまは Loamium 自身がエージェントを内蔵し、右パネルのチャットで対話する。ノートを見ながら質問でき、応答は SSE でストリーミング表示される。
- **接続先の LLM を自分で選べる。** vault の `.loamium/agent.json` に OpenAI 互換 / Anthropic 互換エンドポイントの URL・モデル・API キー(`$ENV_VAR` 参照可)を書くだけ。ローカル LLM(Ollama 等)も可。未設定時はチャット欄の代わりに設定手順ガイドが出る。設定はセッション作成時に遅延読込されるのでサーバー再起動不要。
- **エージェントが vault を検索・参照して出典付きで答える。** 内部で search / query(DQL)/ read_note / backlinks / tags の5ツールを使い、実行の様子はチャット内にツールチップ(実行中はスピナー)で見える。回答内の `[[リンク]]` はクリックでそのノートへ遷移し、存在しないノートは赤で表示される。
- **エージェントは vault の読み取りしかできない。** シェル実行・任意ファイルアクセス・ノート書き込みは一切渡していない(書き込みは次スプリント)。全ツールは vault ルート外へのパスを拒否する。
- **セッションは `.loamium/agent-sessions/` に JSONL で永続化。** ペインを開き直す/リロードすると直近セッションの続きから対話でき、新規セッションも開始できる。
- **ランタイムが Node 22 LTS になった。** 旧ターミナル関連の依存(node-pty / xterm)・コード・設定・`/api/terminal` は完全撤去。`/api/health` から `terminal` フィールドが消え、代わりに `agent: {enabled, reason}` が入った。

## Why this way

- **pi SDK をサーバー内蔵(in-process)** を採用(ADR-0007)。却下: pi をサブプロセス起動 — ツール定義が外出しになり「bash を絶対渡さない」の強制と監査が遠回りになる。却下: 自前ループ — セッション永続化やコンパクションを再発明する量が実は多い。pi SDK なら自作はチャット UI だけで済み、ツール・権限・監査が最短で配線できる。
- **ツール=Loamium 読み取り API のみ、書き込みは次スプリント**(ADR-0008)。エージェントに渡すツール境界がそのままセキュリティ境界。第1リリースは読み取り専用にして権限リスクをゼロにし、先に価値を出す判断(DESIGN 原則2「データ安全性 > 開発速度」)。Web アクセスも第1リリースでは持たせない(漏洩リスクを扱う ADR が別途必要)。
- **接続設定はファイル + 環境変数のみ(設定 UI なし)** — DESIGN 原則5「シンプルさ > 汎用性」。`.loamium/agent.json` は API キー混入リスクがあるため Git 追跡しない。
- **カスタム read ツールを `read_note` に改名 + `excludeTools` で組み込み全排除** — verifier 指摘への対応。pi の組み込み `read`(任意ファイル読取)との名前衝突を排除し、ADR-0008 の境界を pi 内部実装の変更に対して堅牢化した。
- 実装は 3 Story を直列(1 基盤 → 2 接続+チャット → 3 ツール)で進め、worktree 分離は並列性がないため使わず in-place 実行した(decisions.json 参照)。

## What to verify

## What was assumed

- **pi の JSONL セッションを唯一の永続ソースとし、セッション一覧のタイトルは初回ユーザー発言から生成すると仮定。** 大量セッション時の一覧 UX(検索・削除)は未実装 — 必要なら別 Story。
- **接続失敗時は自動リトライを無効化し即エラー表示すると仮定**(pi は ECONNREFUSED を retryable 扱いするため無効化した)。リトライ方針をユーザーに委ねる前提。要件があれば設定化。
- **実行環境の Node は nvm + `~/.local/bin` symlink で 22 を解決すると仮定。** システムの `/usr/bin/node` は 20 のまま。別マシン/CI では `.nvmrc`(22)に従って各自 Node 22 を用意する必要がある。
- **SSE の改行区切りは `\n\n`(Hono の出力に合わせ `\n` split)と仮定。** 標準の `\r\n` は使っていない — 実運用のプロキシ次第で見直す可能性。
- **無人実行の上限(最大ターン数・タイムアウト)は次の定期実行スプリント(S2fe109)で導入する前提** で、対話チャットには明示上限を置いていない。
