Story Sf2f114-2: when:/when-not: 条件付きステップ実行 (ADR-0010)
=================================================================
Date: 2026-07-12

## Summary

Added optional conditional execution (`when` / `when-not`) to all smart-command
step kinds (journal-append / note-append / note-create / template-instantiate).

## Changes

### packages/shared/src/loamium-command.ts
- Added `evaluateCondition(value: string): boolean` export (ADR-0010 truthiness rule)
  - falsey: '' (empty / whitespace-only after trim) / 'false' / '0'
  - truthy: everything else
- Added `stepConditionFields` const with `when?: string` and `'when-not'?: string`
- Spread `stepConditionFields` into all 4 step schemas (additive — no breaking change)

### packages/shared/src/schemas.ts
- Added optional `skipped?: boolean` to `commandStepResultSchema` (additive)
  - ok:true + skipped:true = conditional skip, no side-effect, not a failure

### packages/shared/src/index.ts
- Exported `evaluateCondition`

### packages/server/src/routes/commands.ts
- Imported `evaluateCondition`
- Added condition evaluation block before each step's side-effect try/catch:
  - Expands `when` / `when-not` via `resolveTemplate`
  - Treats unresolved variables (missing.length > 0) as empty string (= falsey)
  - Pushes `{kind, ok:true, skipped:true}` and `continue`s on skip (no fail-stop)
  - Condition expansion errors → step failure + break (defensive)

## Truthiness rule (AC-Sf2f114-2-1)
Falsey literals: empty string after trim / "false" / "0"
Truthy: anything else (including "true", "1", "yes", arbitrary text)
Absent param (unresolved `{{var}}` template): treated as empty → falsey

## JSON contract keys
- `when` (no change to step schema — ordinary key)
- `when-not` (hyphen key — valid YAML, quoted in zod schema as `'when-not'`)
