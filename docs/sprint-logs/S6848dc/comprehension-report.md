## What changed

- **折り返しがぶら下げになった(#1)**: リスト項目のテキストが折り返すと、2 行目以降が左端でなく前行のテキスト開始位置にそろう。ネスト深さに応じてぶら下げ位置も右へずれる。表示のみでファイルは不変。
- **リスト内の画像が表示される(#2)**: リスト項目内の `![](path)` がインライン画像として描画される。※ 実は現行コードで既に描画されており(先行修正で解消済み)、本 Sprint では回帰テストで挙動を固定した(コード改修なし)。
- **複数行の一括レベル変更(#3)**: 複数のリスト行を選択して Tab / Shift+Tab を押すと、選択した全項目(と子)がまとめて 1 段深く/浅くなる。従来は選択末端の 1 項目だけだった。
- **リスト外 Tab がタブ挿入に(#4)**: 段落など非リスト行で Tab を押すとフォーカスがプロパティ欄へ逃げず、素のタブ(4 スペース)が入る。リスト行の従来のインデントは維持。
- **番号付きリストのネスト採番が正しくなった(#5)**: `1.` の下にインデントで作った子が `1.` から再開し、親は連番を維持する。番号は実ファイルの Markdown として正しく書き換わる(見た目だけのごまかしではない)。
- **リストタイプの一括変換(#6)**: 選択したリストを箇条書き ⇄ 番号付きに変換する。Ctrl-K 統一パレットの 2 コマンド + `Ctrl+Shift+7`(番号)/`Ctrl+Shift+8`(箇条書き)。エージェントも `note_convert_list` ツールで同機能を操作できる(監査ログに残る)。

## Why this way

- **採番は単一の updateListener に集約(#5)**、純関数を `packages/shared` に一元化。全編集経路(Tab/Enter/削除/貼付/変換)を一箇所で整合させ二重管理を排除。ピュア Markdown 絶対に沿って装飾でごまかさず実ファイルの番号を書き換える。
- **変換ロジックも `packages/shared` に一元化し UI とエージェントで共有(#6)**。エージェントツールは監査済みサービス層(upsertNote)経由(ADR-0016)、Ctrl-K パレット(ADR-0019)、ケーパビリティ制御(ADR-0015)。却下: UI 専用の変換とサーバ専用の変換を別々に実装 —— 二重管理になるため。
- **#4 は S9ab6c3 の境界を維持しつつフォールバックだけ変更**。AC-S9ab6c3-1-2「非リスト行では Tab を〈リストのインデント操作〉にしない」は守り、フォーカス移動 → タブ挿入 に変えるだけ(プロトタイプで両立を明示・承認済み)。
- **#1 は `.cm-line` スコープのライン装飾 + CSS**。`.cm-scroller` の padding-inline は触らず、ブロックウィジェットの横スクロール破綻(既知の読取幅キャップ落とし穴)を回避。
- **#7 リモート自動競合マージは分離**。現行の last-write-wins 原則(DESIGN_PRINCIPLES priority2)を更新する load-bearing 判断のため、ADR-0030(tentative)を先に起草し実装は次 Sprint(S2df65d)へ。
