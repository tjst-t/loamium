verify-source-endpoints — GET/PUT /api/commands/{id}/source
==============================================================
date: 2026-07-13

## 修正概要

### 根本原因
notes API (`GET/PUT /api/notes/{path}`) が `normalizeVaultPath` を使い .md を強制するため、
`commands/create-todo.yaml` を `commands/create-todo.yaml.md` として検索 → 404 になっていた。

### 追加修正 (本フェーズで発見)
ルーター (`packages/ui/src/router.ts`) の `parseLocation` が、
URL `/n/commands/create-todo.yaml` を `commands/create-todo.yaml.md` に
変換していた (.md を補完するロジックが .yaml/.yml も対象にしていた)。
→ `hasExt = /\.[a-z0-9]+$/i.test(decoded)` で既に拡張子がある場合は .md を付与しないよう修正。

## エンドポイント仕様

GET /api/commands/{id}/source
  → 200: {id, path, content, mtime}
  → 404: {error:'not_found'} — ファイルなし
  → 400: {error:'invalid_path'} — path traversal

PUT /api/commands/{id}/source
  body: {content: string, mtime?: number}
  → 200: {id, path, created, mtime}
  → 409: {error:'conflict'} — stale mtime
  → 403: {error:'forbidden'} — read-only / append-only モード
  → 400: {error:'invalid_path'}
