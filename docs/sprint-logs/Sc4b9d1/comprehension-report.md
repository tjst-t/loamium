# Comprehension Report — エージェント基盤の完成 (Sprints S5a66e4 / S8a3f2e / Sc4b9d1)

_マイルストーン到達時に生成。`autopilot review` の前に読んでください。_

## How to run it

- `make serve` → UI(`make serve-ui` のポート / 既定は portman リース)。API + UI がバックグラウンド起動。停止は `make stop`。
- 開発 vault は `dev-vault/`(git 管理外)。
- 今回の見どころを触る導線:
  - **エージェントチャット**(右サイドバー): 「このvaultのTODOをまとめて」等を投げると、新ツール（スマートフォルダ/コマンド/テンプレート操作）が使われる。ツール実行はチップで可視化。
  - **設定 → エージェント接続**: 「バックエンド」を「外部 API キー ↔ ローカルモデル」で手動切替。ローカル選択時は `.loamium/models/llm/` のモデル一覧（ラジオで1つ選択）・DL・削除・リロード。
  - **コマンドエディタ**: YAML の `kind:` 補完に `agent-run`（7種目）。選択で prompt 中心の雛形挿入。
  - 同梱サンプル `samples/commands/meeting-summary.yaml`（`make samples` で vault の `system/commands/` へ配布）。

> ⚠️ 注意: 実サーバー e2e の CommandEditor / create-todo 系が **既存破損**（このバッチ以前から）。コマンドエディタを実ブラウザで開く導線は不安定な可能性あり。REST/API・エージェントツール経路は健全。

## What changed

- **エージェントが Loamium のほぼ全機能を操作できるようになった。** これまでエージェントは読み取り＋ノート書き込みまでだったが、スマートフォルダ（作成/取得/更新/削除）・スマートコマンド（一覧/実行）・テンプレート（一覧/適用してノート生成）を操作するツールが加わった。各ツールは REST と同じ監査済みサービス層を通り、権限（ケーパビリティ）と機密領域 deny を継承する。
- **スマートコマンドから AI エージェントを呼べる `agent-run` ステップが追加。** 議事録まとめのような「賢い処理」を宣言的コマンドの1ステップとして起動できる。既存の非同期ジョブ基盤（S2fe109）を再利用し、結果はエージェント自身の書き込みツールで vault に残す（独自の書き込み経路は作らない）。
- **外部 API なしで動く内蔵オフライン LLM が入った。** node-llama-cpp をサーバー内に組み込み、OpenAI 互換の内部 shim（`/api/llm/v1/chat/completions`）を通してエージェントの推論バックエンドをローカルモデルに切り替えられる。モデルは `.loamium/models/llm/`（将来の音声認識は `.loamium/models/asr/`）に置き、UI から DL/一覧/削除できる。
- **バックエンド選択はユーザーの明示操作。** API キーが無い/オフラインでも自動でローカルへ切り替わらない。ユーザーが設定で「外部」か「ローカル」を選び、選んだ側が未準備ならエージェント接続は無効（プロトタイプレビューでの要望を反映）。
- **CLAUDE.md に規約追記**: 「新機能を足すときは必ずエージェント操作ツールも実装する」（監査済みサービス層経由・help に使い方追記）。

## Why this way

- **agent-run は既存 runAgentJob(S2fe109) を再利用**（却下: 新規ジョブ実行基盤 — 実行経路が2本になり監査・権限・上限が二重化）。→ ADR-0028 を新規作成。結果書き込みは prompt がエージェントの journal_append/note_create 等に委譲（却下: agent-run 独自 I/O — ピュア Markdown 保証・監査が二重管理になる）。
- **内蔵 LLM は in-process + OpenAI 互換 shim**（ADR-0025）。pi の baseUrl を内部 shim に向けるだけで既存のエージェント/ツール/監査配線を無改造で再利用。エンジンは shim の裏で差し替え可能。
- **バックエンドは明示選択・非フォールバック / モデルは種別サブフォルダ**（ADR-0025 amendment, 2026-07-16, ユーザー主導）。却下: 自動フォールバック優先度 — オフライン時の挙動が予測不能になる、を嫌った。
- **エージェントツールは既存サービス層を純関数へ切り出して共有**（ADR-0016）。REST とツールが同一ロジックを通り、ピュア Markdown・パス正規化・監査・機密 deny を自動継承。エージェント専用の直接ファイル操作は新設しない。
- **モデル管理は REST のみ**（CLI 省略）。priority 3(REST/CLI 1:1) の核は journal/search でモデル管理はコア外、と判断した限定的逸脱（compromises.json 参照）。

## What to verify

- ⚠️ **(high, 検証者が捕捉→修正済み)** `command_run` / `template_instantiate` の書込先に機密 deny が効いていなかった（AC-2-3/3-3 違反）。fix 2123897 で agent 経路のみ deny 強制＋回帰テスト追加。**機密領域（deny 対象）へエージェントがコマンド/テンプレート経由で書けないこと**を実際に触って確認推奨。
- ⚠️ **(medium, 環境制約)** 実 GGUF での推論・実モデル DL は未実証（スタブ検証）。**実 VM でモデルを1つ DL → ローカルバックエンド選択 → エージェントがオフラインで応答するスモーク**が要フォローアップ。ネイティブビルドは gcc-13 ソースビルドで通ることを確認済み（[[node-llama-cpp-build]]）。
- ⚠️ **(medium, 既存破損)** CommandEditor / create-todo 系の実サーバー e2e が失敗中（このバッチ以前から）。コマンドエディタの実ブラウザ動作を確認し、agent-run 補完も含め崩れがないか目視。
- 設定の「外部 ↔ ローカル」切替 UI、モデル一覧（0件時の空表示・リロード）、DL 進捗の見た目（プロトタイプ承認済みだが実 UI で最終確認）。
- ストリーミング応答が単一 delta（一気に出る）で違和感がないか。

## What was assumed

- **実推論の品質・速度は未検証**。CPU の 3-4B Q4 モデルで実用に足るかは実機依存。ADR-0025 は「外部 API の代替でなく『無いよりまし』」と位置付け。
- **モデル DL の実 HTTP・進捗**はスタブで検証。実大容量 DL のタイムアウト/再開/帯域挙動は未確認。
- **multi-turn 忠実度**: エンジン面が完了文字列のみ返すため、会話履歴を System/User/Assistant ラベル連結でプロンプト縮約している。長い会話の忠実度は落ちうる。
- **agent-run の議事録まとめ**は実 Pi ツール経路での二重監査（command.run + agent 書き込み）を未実証（スタブ検証）。実 LLM を起動する end-to-end は CI 方針上テストしていない。
