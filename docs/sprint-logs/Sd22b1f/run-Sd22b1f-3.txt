== resolveTemplate missing-optional-var convention ==
When `{{due}}` is in a content template and `due` is absent from params,
resolveTemplate leaves the token verbatim as `{{due}}` (collected in missing[]).
Convention for create-todo-section fixture: `due` is NOT included in the
journal-append content template ("- [ ] {{summary}}"), so the line is clean
regardless of whether due is provided. A separate fixture (create-todo-with-due)
documents the verbatim-token behavior explicitly in test code with assertions.

== CLI dash-content convention ==
Positional content starting with "-" must be passed after the "--" end-of-options
separator. This is the standard POSIX/Commander convention:
  loamium journal-append --section Todo -- "- [ ] task"
  loamium journal-append -- "- [ ] task" 2026-01-01
The <content> positional interface is NOT changed. The "--" is purely an invocation
convention required when content begins with "-".
