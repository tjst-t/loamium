# Comprehension Report — レビュー修正 第3ラウンド (Sprint Sd40b63)

_Generated at milestone arrival. Read this before `autopilot review`._

実機フィードバック3件(テーブルの WYSIWYG 編集・`/` のテーブルサイズ指定・`/` のスクロール追従)の修正バッチです。

## What changed

- **Markdown テーブルを表として直接編集できるようになった。** ライブプレビュー上のテーブルは、セルをクリックして中身をその場で書き換えられ、フォーカスを外すと表として再描画されます。行の追加・削除、列の追加・削除もボタン(行末の「+ 行追加」・列末の +・セルの ×)でできます。**編集内容は常に標準 Markdown テーブル**(ヘッダ + `---` 区切り + データ行)としてファイルに書き戻されるので、Obsidian や素のエディタで開いても壊れません。
- **`/` メニューのテーブル挿入で列数×行数を選べるようになった。** 「テーブル」を選ぶと **Notion 風のグリッドピッカー**(ホバー/矢印キーで N×M、既定 3×3、最大 8×8)が出て、選んだサイズの標準 Markdown テーブルが挿入され、カーソルが先頭セルに入ります。
- **`/` メニューのキーボード選択でスクロールが追従するようになった。** ↑↓ で項目を辿ると、選択中の項目が常に表示範囲に入るようスクロールします(下端の項目が画面外に隠れるバグを修正)。

## Why this way

- **WYSIWYG は表示・編集層だけ、ファイルは常に標準 Markdown**(priority 1: ピュア Markdown 正本性)。セルの `|` は `\|` にエスケープして往復し(可逆であることをユニットテストで検証)、独自記法・ブロック ID・不可視文字は一切入れません。
- **セルは contenteditable ではなく `<input>`**(自律判断)。CodeMirror がウィジェット内のキャレット/選択を横取りするのを避けるため。IME 入力も壊れません。
- **セル編集の確定は `blur`(relatedTarget 判定)で行う**。実機 E2E で `focusout` が `blur` より先に発火して編集が取りこぼされる問題を見つけ、順序に依存しない形にしました。
- **グリッドピッカーの数値『任意入力』(8×8 超)とセル内改行は先送り**(AC はホバー/キーの N×M グリッドで充足。ユーザー要望どおり)。

## What to verify

- ⚠️ **以前『テーブルが生の `|` のまま』に見えた件**は、カーソルがテーブル上にあった(=ソース表示)か、ブラウザが古いバンドルをキャッシュしていた可能性が高いです。今回の WYSIWYG ウィジェットはそれを包含します。反映されない場合は**ブラウザのハードリロード(Cmd/Ctrl+Shift+R)**を試してください。実機デモの描画確認は `docs/sprint-logs/Sd40b63/demo/`。
- テーブルのセル編集 → フォーカスアウトで再描画 → ファイルが標準 Markdown になっているか、`/` のグリッドピッカーで好きなサイズを挿入できるか、`/` メニューを↑↓で辿って下端項目が隠れないかを実機で確認してください。
- ⚠️ (既存 flaky・要保守) `tests/acceptance/terminal.spec.ts` の Origin 拒否監査テストが、フルスイート並列実行で稀に落ちます(WS 監査ログの flush race。単独では 13/13 pass、今回の変更とは無関係)。テストの決定性を上げる保守を backlog 化しました。

## What was assumed

- テーブルのセルは 1 行テキスト(セル内改行なし)。列幅パディングは最小(整形しすぎない)。区切り行は `---`(整列記号 `:---:` は将来対応でも可)。
- グリッドピッカーは最大 8×8。それ以上は挿入後に行/列追加ボタンで増やす想定。
- WYSIWYG 編集はライブプレビュー(カーソル行以外を装飾)の枠組みの上に載る。テーブル行にカーソルがあるときの挙動は「表として編集」を優先。
