# Comprehension Report — 第3バッチ / UI シェル刷新 (Sprints Sf1a90a, S935867, Seac77a, S763a98)

_Generated at milestone arrival. Read this before `autopilot review`._

これはマイルストーンレビューでのあなたのフィードバック(タブ廃止・戻る/進む・Claude を右サイドバー・検索の2モード・アセット一覧・スラッシュメニュー)を実装したバッチです。

## What changed

- **UI がタブ方式から「ブラウザ的な1画面ナビゲーション」に変わった。** 上部のタブは廃止。ノート・検索・ファイル一覧はそれぞれ URL を持つルート(/n/{path}・/search・/files)になり、**ブラウザとヘッダの戻る/進むボタン**で辿れる。リロードしても同じノートに戻る。[[リンク]]・バックリンク・検索結果・embed からの移動もすべて履歴に積まれる。
- **Claude(ターミナル)が右サイドバーのトグルになった。** 右サイドバーは「バックリンク ⇄ Claude」の切り替えになり、**Claude と対話しながらメインのノートが見える**。タブ時代の「1画面しか見られない」問題を解消。切り替えても xterm セッションは維持される。
- **報告いただいた『claude が動かない』を修正した。** node-pty へ渡す環境変数から `CLAUDECODE` 等を除き、`TERM`/`COLORTERM`/`LANG` を整えた。実サーバーの手動 smoke で claude が「Do you trust this folder?」プロンプトまで到達することを確認済み。
- **検索が2モードになった。** 素早いジャンプ用の Cmd+K ポップアップ(1件で閉じる)はそのまま、腰を据えて探すための**詳細検索ページ /search** を追加。全文・タグ・フォルダ・並び順を指定でき、**結果一覧を開いたまま複数の結果を順に見られる**。検索条件は URL に入るのでブックマーク・戻る/進むで再現でき、最近の検索履歴も残る。
- **ファイル/フォルダブラウザ /files ができた。** フォルダを辿ってノートも添付ファイルも一覧でき、名前で絞り込み・プレビュー・**vault 相対パスのコピー**・削除ができる。サイドバーは直近10件に絞られ、「すべて表示」からここへ来る。
- **`/` スラッシュメニューができた。** エディタで `/` を打つとメニューが出て、テーブル・callout・コードフェンス・mermaid・dataview・チェックボックス・見出し・日付を暗記なしで挿入できる。**挿入結果はすべて標準 Markdown**。コードフェンス内・インラインコード内では発火しない。

## Why this way

- **ナビゲーションはブラウザ的ルーティングに統一**(却下: タブ維持 — あなたが「タブは開きすぎる」と明言、DESIGN_PRINCIPLES に新原則として記録)。メインは常に1画面、機能はルートとして遷移する。
- **Claude はメインを占有させず右サイドバー**(あなたの提案どおり)。ノートを見ながら対話が中核価値のため。
- **検索は「タブ」ではなく「ルート」**。ポップアップは残し、詳細検索は別ルートにして2モードを共存(ポップアップに『詳細検索を開く』導線)。
- **Seac77a を『アセット専用』から『vault 全体ブラウザ』へ拡張**(却下: アセットのみ — シェル刷新でサイドバーのノートフォルダ閲覧が消えた分を、この /files で受ける必要があった。verifier の指摘を受けマージ前に ROADMAP を修正)。
- **スラッシュメニューの挿入は標準 Markdown のみ**(priority 1: ピュア Markdown 正本性。独自記法は作らない。既存の3レジストリが描画できる記法に揃えた)。
