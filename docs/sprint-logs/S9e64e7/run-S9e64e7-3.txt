## section/target 補完の方針 (ドキュメント)

- section: 補完 → 静的ヒント (Todo/Done/Notes/Tasks/Log/Journal/Ideas/Archive)
  理由: ノートの見出し一覧をリアルタイム取得するには GET /api/notes/{path}/meta が必要で、
  current document path が CommandEditor に伝播されていないため Live fetch は過剰。
  静的ヒントで "入力しやすさ" (ADR-0011) の要件を満たす。

- target: 補完 → マウント時に GET /api/notes を 1 回フェッチしてキャッシュ (vault ノートパス一覧)
  理由: gui-spec endpoint_contracts に "GET /api/notes … NoteListResponse を補完候補として使用"
  と明記されているため実装。フェッチ失敗は補完なし (フォールバック)。
  キャッシュは setCommandDslNotePaths() でテスト注入可能。

## 技術メモ: CodeMirror interactionDelay

CodeMirror の acceptCompletion は "timestamp から 75ms 未満" の場合に return false する
(誤確定防止の安全策)。Playwright テストで popup 出現直後に Enter を押すと 75ms 未満の
可能性があるため、allTextContents() などの DOM クエリが 75ms 以上かかる場合は自然に
解消されるが、短いクエリ (1 文字) の場合は waitForTimeout(200) を挟む必要がある。
tests 8/9/11/12 で waitForTimeout(200) を使用。
