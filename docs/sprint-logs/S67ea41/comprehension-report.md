# Comprehension Report — プロパティUI改訂 + テンプレートシステム (Sprints Sd13ab1, S89a350, S67ea41)

_Generated at milestone arrival. Read this before `autopilot review`._

プロパティUIのレビュー反映と、(別セッションから復元した)テンプレートシステムを実装したバッチです。

## What changed

**プロパティUI(Sd13ab1)**
- **畳み時に値の要約バー**が出るようになった(`> #sample-book #science · ●読了 · ★★★★☆ · 📅2026-05-20 · +6`)。「プロパティ」というラベル語は出しません。
- **追加フローをキーファーストに作り直した。** 「+ プロパティを追加」は**キー候補メニュー**になり、①既知/一意プロパティ(tags/status/rating… + **vault 内で使われている全キー**)は選ぶだけで即追加(既存キーは無効)、②新しい名前は「新規作成」で型を選んで追加。型ファーストの不整合(tags を選んだのにキー名を再度聞く)を解消。
- **他ファイルで作ったキーが別ファイルでもサジェスト**される(`GET /api/property-keys`、件数付き・chokidar 追従)。**新規プロパティで選んだ型は `.loamium/property-types.json` に永続化**され、そのキーは全ファイルで同じ型に解決(型はプロパティ名にひもづく=Obsidian/Notion 相当)。
- **行削除UI**(各行ホバーの ×)と、**frontmatter 無しノートへの追加入口**が付いた。

**テンプレートシステム(S89a350)**
- vault の `templates/` 配下の Markdown をテンプレートとして扱う。frontmatter の `loamium-template.target`(保存先パターン)と `vars`(変数定義)で駆動。`{{name}}` と `{{date:YYYY-MM-DD}}` 等を解決。
- **「テンプレートから新規作成」→変数入力モーダル**(キーボード完結、必須検証、保存先ライブプレビュー)→ 解決先パス(衝突時は連番)にノート作成。`POST /api/templates/{name}/instantiate` と CLI `loamium new --template <名> --var k=v` も 1:1。
- **生成ノートは解決済みピュア Markdown**(`{{...}}`・テンプレ記法は結果に残らない)。

**journal のテンプレート化(S67ea41)**
- 日次ジャーナルの**遅延生成**(初回アクセスで作る挙動)は維持したまま、`templates/journal.md` があればそれを適用した本文で作る。**無ければ従来通り空ファイル(後方互換)**。`{{date:...}}` は対象日基準(過去/未来日も)。read-only では書かず適用済みの仮想ジャーナルを返す。既存は上書きしない冪等。

## Why this way

- **キーファースト**は、S87f4b7 の **D方式(型はキーから決まる)** と整合させるため。主役はプロパティ名で、型はそれに従う。あなたの「型=キーだと思っていた」という直感どおりです。
- **型情報・キー集約は `.loamium/`**(使い捨て領域)。ノート本文は `rating: 4` の素の YAML のまま(priority 1・6、ロックインなし)。
- テンプレートの在り処は**新しい設定ファイルを増やさず** `templates/` + `loamium-template` frontmatter に統一(S89a350 の規約を journal でも再利用)。作成は**遅延生成のまま**(プロアクティブ cron は採らない)。
- テンプレ記法は**テンプレートファイル内のみ**で、生成結果には残さない(独自記法をユーザーの通常ノートに入れない)。

## What to verify

- ⚠️ **要承認: テンプレート変数入力モーダル(S89a350-3)は needs_user_review** です。全 AC(キーボード完結・必須検証・保存先ライブプレビュー・作成→エディタで開く)は実装・E2E 済みですが、**UX は実機で見て承認してほしい**項目です。承認すれば S89a350 が partial→done、進捗が **24/24(100%)** になります。実機で「テンプレートから新規作成」を触ってみてください。
- 実機確認(**ハードリロード推奨**): プロパティの値要約バー / キーファースト追加(他ファイルのキーがサジェスト・型の横断固定)/ 行削除 / 空ノート追加 / テンプレートからの新規作成 / journal がテンプレ適用で開く。
- ⚠️ 日付フォーマットは YYYY/MM/DD/HH/mm/ss のみ。**曜日(ddd 等)は未対応**でリテラルのまま残ります(要れば backlog)。
- 複数回再発していた `terminal.spec.ts` の flaky をこの機に安定化しました(ポーリング化、アサーション不変。516/516 を2回連続確認)。

## What was assumed

- 既定 journal テンプレートは `templates/journal.md`(あれば)。無ければ空ファイル。
- テンプレートの変数型は property-types と同じ(text/select/date/tags)。パス用途の変数値はサニタイズ(`/`・`..`・制御文字)してから normalizeVaultPath を通す。任意 eval なし。
- 新規プロパティの型永続化先は `.loamium/property-types.json`(vault ローカル、gitignore 対象)。
- テンプレートシステムの設計は、同時に走っていた別 Claude セッションの計画をトランスクリプトから復元したもの(内容はそのセッションの意図どおり)。
